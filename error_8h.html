◆ REPROC_ERROR

enum REPROC_ERROR

Error enum. Contains all errors that may be returned from reproc's API. Because REPROC_SUCCESS evaluates to zero it is possible to check for errors as follows:

REPROC_ERROR error = reproc_read(...);
if (error) { return error; } // Only executes if reproc_read returns an error
Enumerator
REPROC_SUCCESS 

Indicates a library call was successful. All library function return this value if no error occurs.

REPROC_WAIT_TIMEOUT 

A timeout value passed to a function expired.

REPROC_STREAM_CLOSED 

The child process closed one of its streams (and in case of stdout/stderr all of the data from that stream has been read).

REPROC_PARTIAL_WRITE 

Only part of the buffer was written to the stdin of the child process.

REPROC_NOT_ENOUGH_MEMORY 

A memory allocation in the library code failed (Windows only) or the underlying system did not have enough memory to execute a system call.

REPROC_PIPE_LIMIT_REACHED 

The current or child process was not allowed to create any more pipes.

REPROC_INTERRUPTED 

A waiting system call (read, write, wait, ...) was interrupted by the system.

REPROC_PROCESS_LIMIT_REACHED 

The current process was not allowed to spawn any more child processes.

REPROC_INVALID_UNICODE 

One of the UTF-8 strings passed to the library did not contain valid unicode.

REPROC_PERMISSION_DENIED 

The current process does not have permission to execute the program.

REPROC_SYMLINK_LOOP 

Too many symlinks were encountered while looking for the program.

REPROC_FILE_NOT_FOUND 

The program or working directory was not found.

REPROC_NAME_TOO_LONG 

The given name was too long (most systems have path length limits).

REPROC_UNKNOWN_ERROR 

Unlike POSIX, Windows does not include information about exactly which errors can occur in its documentation. If an error occurs that is not known functions will return REPROC_UNKNOWN_ERROR.

◆ reproc_system_error()

unsigned int reproc_system_error ( void  )

Returns the last system error code.

On Windows this function returns the result of GetLastError. On POSIX platforms this function returns the value of errno. The value is not stored so other functions that modify the results of GetLastError or errno should not be called if you want to retrieve the last system error that occurred in one of reproc's functions.

On POSIX, if an error occurs after fork but before exec it is communicated to the parent process which sets its own errno value to the errno value of the child process. This makes it possible to retrieve errors that happen after forking with this function (for example in chdir or execve).
